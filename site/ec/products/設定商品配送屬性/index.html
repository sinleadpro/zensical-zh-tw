設定商品配送屬性¶

綁定商品可使用的宅配物流、溫層與出貨通路，含一般宅配與宅配貨到付款

[ 相關操作]{.badge-icon} [[批次修改商品溫層/配送方式/通路]]
[ 適用方案]{.badge-icon} 專業｜進階｜高手｜企業

說明概述¶

  • 不同配送屬性（物流／溫層／通路）商品將自動[[設定多購物車#多購物車結帳說明|拆分購物車]]並分別計算運費與優惠
  • 新增物流後，請檢查**僅綁定特定物流的商品**是否漏綁
  • 一般宅配與貨到付款物流皆支援多溫層（常溫／冷藏／冷凍）
  • 指定黑貓／宅配通等物流配送
  • 指定溫層配送（例如冷凍品）
  • 指定出貨通路（例如台北倉、台中倉、門市）

操作流程¶

步驟一：設定宅配運費規則¶

含一般宅配與貨到付款兩類運費設定

A. 一般宅配運費設定¶

後台路徑：金物流 → 宅配運費設定
- 點擊 新增運費規則 - 設定物流名稱、溫層、消費門檻、運費、付款類型 - 若有多段門檻，重複新增規則

注意

  • 系統已內建「黑貓貨到付款」「宅配通貨到付款」 → 避免建立同名物流
  • 同物流不同溫層請使用不同名稱（例：黑貓-冷藏、黑貓-冷凍）

B. 貨到付款運費設定¶

後台路徑：金物流 → 宅配物流
以黑貓貨到付款為例：

  • 開啟 貨到付款
  • 可設定 免運門檻
  • 設定不同溫層運費
    以代收金額為運費基準

Warning

若代收金額 > 免運門檻 → 該訂單視同免運

步驟二：綁定商品配送溫層¶

後台路徑：商品 → 所有商品 → 編輯商品 → 設定

  • 勾選商品可用溫層（可複選）
  • 預設為常溫

步驟三：綁定商品可用物流¶

同頁面 溫層與物流設定

  • ON = 商品可用此物流
  • OFF = 不適用
  • 預設商品**可用所有物流**，至少需保留一項

Tip

首次修改會跳提醒 → 必須勾選「確認」後才會不再彈出

步驟四：設定出貨通路（選填）¶

若商品從不同倉別／門市出貨，可綁定出貨通路

建議僅必要時設定，多通路＝多購物車

管理注意事項¶

  • 原設定**全部物流**的商品 → 自動綁定新物流
  • 原設定**指定物流**的商品 → 不自動綁定，需人工處理
  • 重新檢查物流溫層與商品綁定設定
  • 原商品綁定會失效 → 需重新綁定

使用情境範例¶

商品 原設定 新增物流後結果
A 適用全部宅配 ✅ 自動綁定
B 只綁黑貓 ❌ 不自動綁新竹物流

Tip

新增物流後 → 請檢查「只綁部分物流」商品

前台購物車體驗¶

若購物車中商品配送屬性不同：

  • 系統自動分成多個購物車
  • 每車獨立起算運費與優惠
  • 顧客需多次結帳

最佳實務

越多配送屬性 = 越高棄單風險
建議簡化配送設定以提升結帳成功率

常見問題¶

可以設定同商品多溫層嗎？

可以，支援複選

修改物流名稱後為什麼商品配送變不見？

物流綁定依名稱比對 → 名稱變更等於新物流 → 需重新綁定

為什麼會出現多個購物車？

因商品配送或通路設定不同
→ 參考 [[設定多購物車#多購物車結帳說明]]
