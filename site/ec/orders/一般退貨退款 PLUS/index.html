PLUS用戶 - 一般退貨退款

功能說明：

  • 訂單商品須進行退貨、退款之操作流程。
  • 商家及消費者退貨流程說明。

操作目錄：

1.訂單退款形式說明(非 LINE PAY、街口支付)
A : 商家操作退款流程

  • 可【自動退款】訂單

  • 需【人工退款】訂單 B : 顧客申請退貨退款

  • 顧客操作

  • 商家操作 ------------------------------
    2.訂單退款形式說明(LINE PAY、街口支付)

  • 方法一 : 顧客前台點選申請退貨

  • 方法二 : 商家自行操作退貨 ------------------------------
    3.消費者可於前台申請退貨時間

注意事項:

  • 使用 CYBERBIZ 電商倉儲者，不適用於此文件。(高手版PLUS)
  • 退貨 :
  • 每筆訂單僅接受一次退貨申請。

  • 紅利/優惠券 處理方式
    - 該訂單中使用的紅利/優惠券，系統「不會」自動歸還至顧客帳戶。
    - 該訂單消費所獲得的紅利/優惠券回饋，系統「不會」從顧客帳戶扣除。
    - 如需刪除個人紅利/優惠券，請至「會員」→「所有會員」→ 至顧客個人頁面進行刪除。

  • 分潤處理 - 退貨時，只要曾經「結案」過的訂單，就會計算分潤；退貨後，不影響分潤計算。

  • 退款 :
  • 每筆訂單僅接受一次退款申請。
  • 退款方式依顧客付款方式不同而有所差異。
  • 若需 CYBERBIZ 進行人工退款之訂單，每筆將酌收30元帳務處理費。
  • CYBERBIZ 人工退款將於約 7-10 個工作天退款至顧客指定帳戶。
  • 退款後發票處理狀態(美麗科技、綠界) : 作廢/折讓(開立發票時間 5 號以前為作廢，之後為折讓)。

📌 訂單退款形式說明 (非 LINE PAY、街口支付)¶

支付方式 自動退款 需人工退款 備註
信用卡 ✅ 付款超過180天
Paypal ✅ 付款超過180天
AFTEE ✅ ❌ 自動退款至顧客帳戶
ATOME ✅ ❌ 自動退款至顧客帳戶
貨到付款 ❌ ✅
超商代碼 ❌ ✅
虛擬ATM ❌ ✅

附註事項： 1. 超過期限後，商家需向顧客索取退款帳戶資訊，由CYBERBIZ進行人工退款。 2. 人工退款將酌收 每筆 30 元 帳務處理費。 3. 貨到付款、超商代碼、虛擬ATM：需向顧客索取退款帳戶資訊，由 CYBERBIZ 進行人工退款。

商家操作退款流程¶

一、可【自動退款】訂單：

情境 :

  • 支付方式為 信用卡、Line Pay等可自動退款之訂單。

  • 若訂單已超過商家設定退貨退款時間，需由商家於後台操作。 操作步驟 :

  • 點選要退貨退款的訂單，選擇操作「退貨中」。
    *此狀態選擇後，將會同步顯示於顧客前台訂單狀態。

若商家需派車取回商品，則可至後台操作逆物流將商品取回。 逆物流教學文件

  1. 點選「退貨審查」
    *退貨審查狀態為商家已收到顧客寄回的商品，檢查商品是否完整。

  2. 依照收到的商品，商家自行評估是否退款給消費者。

  3. 收到退貨商品 → 勾選訂單 → 已退貨 → 直接退款消費者

  4. 收到退貨商品 → 勾選訂單 → 拒絕退貨 → 不退款給消費者 (*一旦勾選拒絕退貨就無法再進行退貨退款作業。)

二、需【人工退款】訂單：

情境 :

  • 支付方式為 貨到付款、虛擬ATM..等，不支援自動退款之訂單。
  • 支付時間已超過可自動退刷期限之訂單。
  • 若訂單已超過商家設定退貨退款時間，需由商家於後台操作。

  • 此退貨退款方式需輸入顧客退款資料。 操作步驟 :

  • 點選要退貨退款的訂單，選擇操作「退貨中」。
    *此狀態選擇後，將會同步顯示於顧客前台訂單狀態。

若商家需派車取回商品，則可至後台操作逆物流將商品取回。 逆物流教學文件

  1. 點選「退貨審查」
    *退貨審查狀態為商家已收到顧客寄回的商品，檢查商品是否完整。

  2. 依照收到的商品，商家自行評估是否退款給消費者。

  3. 收到退貨商品 → 勾選訂單 → 已退貨

  4. 收到退貨商品 → 勾選訂單 → 拒絕退貨 → 不退款給消費者 (*一旦勾選拒絕退貨就無法再進行退貨退款作業)

  5. 操作已退貨後，會跳出「人工退款資料」，依照顧客提供的退款資料填寫。
    送出後 CYBERBIZ 將於 7-10 個工作天退款至顧客指定帳戶。

  6. 申請人工退款後，商家後台及顧客畫面。

商家訂單後台： 將顯示退款資訊以及訂單備註

顧客前台： 會員資料中同樣顯示退款資訊

顧客申請退貨退款¶

使用情境 : 訂單並未超過商家設定的退貨退款期限，才會顯示此功能，若已超過需由商家於後台操作。

  • 訂單的支付方式可自動退刷，商家審核後將自動退款。

  • 訂單的支付方式若不支援自動退刷，須輸入顧客退款資料，由 CYBERBIZ 進行人工退款。 顧客操作 :
    後台路徑 : 「訂單查詢」→「我的帳戶」→「近期訂單」

  • 選擇想要退貨的訂單，點選「可退貨」。
    列表中訂單會呈現「可退貨」的狀態，點選欲退貨的訂單編號，進入訂單明細。

  • 點選「申請退貨」

  • 填寫退貨原因及退款資訊並點選「確認退貨」。

  • 訂單的支付方式可自動退刷將自動退款。(如：信用卡、LINE PAY)

  • 訂單的支付方式若不支援自動退刷，須輸入顧客退款資料，由 CYBERBIZ 進行人工退款。
    (如：貨到付款、虛擬ATM)

  1. 顧客於前台申請完成後，商家需進行退款操作。
    *顧客申請完成後，訂單退貨狀態將變更至【退貨申請】

商家操作 :

  1. 點選要退貨退款的訂單，選擇操作「退貨中」。
    *此狀態選擇後，將會同步顯示於顧客前台訂單狀態。

若商家需派車取回商品，則可至後台操作逆物流將商品取回。 逆物流教學文件

  1. 點選「退貨審查」
    *退貨審查狀態為商家已收到顧客寄回的商品，檢查商品是否完整。

  2. 依照收到的商品，商家自行評估是否退款給消費者。

  3. 收到退貨商品 → 勾選訂單 → 已退貨 → 退款作業

  4. 訂單的支付方式可自動退刷將自動退款。
  5. 訂單的支付方式若不支援自動退刷，由 CYBERBIZ 進行人工退款流程。
  6. 收到退貨商品 → 勾選訂單 → 拒絕退貨 → 不退款給消費者 (*一旦勾選拒絕退貨就無法再進行退貨退款作業)

📌 訂單退款形式說明 (LINE PAY、街口支付)¶

LINA PAY、街口支付退款流程 (超過60/180天後操作流程)¶

1️⃣方法一 : 顧客前台點選申請退貨

【顧客端】

  1. 顧客點選「申請退貨」。

  2. 進入退貨申請頁面。

  3. 填寫退貨、退款資料，按下「確認」。

【商家端】

  1. 訂單狀態轉為「退貨申請」。

  2. 勾選訂單後，將狀態改為「退貨中」。

  3. 勾選訂單後，將狀態改為「退貨審查」。

  4. 勾選訂單後，將狀態改為「已退貨」。

  5. 此時訂單付款狀態會是「待退款」。商家匯款給顧客完成退款之後，自行將狀態改為「已退款」。

2️⃣方法二 : 商家自行操作退貨

  1. 勾選訂單後，將狀態改為「退貨中」。

  2. 勾選訂單後，將狀態改為「退貨審查」。

  3. 勾選訂單後，將狀態改為「已退貨」。

  4. 填寫顧客銀行資訊。

  5. 此時訂單付款狀態會是「待退款」。商家匯款給顧客完成退款之後，自行將狀態改為「已退款」。

📌 消費者可於前台申請退貨時間¶

  1. 使用 CYBERBIZ 串接出貨方式(宅配、超取)訂單，可於 10 天內申請退貨
  2. 使用自訂物流出貨(其他方式)訂單，可於 14 天內申請退貨 *超過上述期限，需由商家於後台操作(請至 商家操作流程 查看)
