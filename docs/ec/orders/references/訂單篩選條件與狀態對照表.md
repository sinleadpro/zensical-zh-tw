## 篩選條件總覽 { #orders-v2-filters }                                                       

!!! info "可供選擇的篩選欄位與選項值，將根據您的 **使用方案** 或 **後台功能設定** 而動態調整。"


| 篩選條件 | 說明 | 開通條件 |
|---|---|---|
| **[訂單狀態][orders-status]{ data-preview }** | 依訂單整體流程狀態過濾（進行中／已結案／已取消） | 預設可用 |
| **[付款狀態][orders-financial-status]{ data-preview }** | 依金流收款／退款狀態過濾（已收到款項、等待付款、已退款等） | 預設可用 |
| **[配送狀態][orders-fulfillment-status]{ data-preview }** | 依物流出貨流程狀態過濾（未出貨、已出貨、已到店等） | 預設可用，部分選項需 **物流系統** |
| **[退貨狀態][orders-return-status]{ data-preview }** | 依退貨流程狀態過濾（退貨申請、退貨中、已退貨等） | 需 **物流系統** |
| **[訂單來源][orders-store]{ data-preview }** | 區分 EC 線上訂單與各 POS 實體門市訂單 | 需 **POS 系統** |
| **[商家來源/<br>收款方][orders-merchant-from]{ data-preview }** | 依金流實際收款方過濾（順立代收／商家／POS／倉庫） | 企業版、各 PLUS 版，或未啟用「自有金流」 |
| **[配送方式][orders-shipping-name]{ data-preview }** | 依物流通路過濾（超商店取、宅配、自取等） | 預設可用，部分選項需 **門市取貨 / POS** |
| **商品通路** | 依訂單中的商品通路標記過濾，適用多通路販售情境 | 需 **商品通路設定** |
| **配送溫層** | 依商品綁定的常溫／冷藏／冷凍溫層過濾 | 需 **商品綁定溫層** |
| **訂單標籤** | 依商家自訂的訂單標籤過濾（可於訂單詳情手動新增） | 預設可用 |
| **訂單商品** | 直接選擇特定商品，僅顯示包含該商品的訂單 | 預設可用 |
| **[訂單類型][orders-line-group-buy]{ data-preview }** | 過濾 LINE 團購訂單 | 需 **LINE 團購** |
| **訂單成立日期** | 依訂單建立時間過濾，支援「介於」與「過去 N 天」兩種模式 | 預設可用 |
| **訂單配送日期** | 依預定或實際配送日期過濾，支援「介於」與「過去 N 天」 | 預設可用 |

## 付款狀態對照表 { #orders-financial-status }

| 付款狀態 | 說明 |                                              
| :-- | :-- |                                                    
| 等待付款 | 訂單成立但顧客尚未完成付款，常見於 ATM 轉帳、超商代碼繳費等待繳費期間 |                                 
| 已收到款項 | 顧客已完成付款且款項已入帳 |                      
| 貨到付款 | 顧客選擇貨到付款，款項將於配送時由物流代收 |        
| 已回填帳號未收款 | 顧客已回填匯款帳號後 5 碼，但商家尚未確認入帳 |                                         
| 付款失敗 | 付款流程中斷或被金流方拒絕，例如信用卡授權失敗 |    
| 顧客棄單 | 顧客未完成付款且訂單已逾期作廢 |                    
| 除顧客棄單以外 | 篩選快捷鍵：一次選取所有「非棄單」的訂單 |    
| 客服處理中 | 該訂單因金流／退款問題進入客服人工處理階段 |      
| 待退款 | 已確認需退款但款項尚未退出 |                          
| 退款中 | 退款流程進行中（已送出退款指令，等待金流回應） |      
| 已退款 | 全額退款已完成 |                                      
| 退款失敗 | 退款流程失敗，需人工處理 |
| 待部分退款 | 已確認需部分退款但款項尚未退出 |                  
| 部分退款 | 部分金額退款已完成（例如顧客退掉訂單中部分商品） |
| 不退款 | 訂單已標記為不進行退款（例如顧客同意換貨而非退款） |

| 已出貨 | 商家已將訂單交付物流，包裹在運送途中 |

| 已到店 | 超商取貨訂單已抵達指定門市，等待顧客取件 |            
| 已收貨 | 顧客已完成簽收 |                                      
| 逾期未取 | 超商取貨訂單超過保留期限未被取走，已退回商家 |
| 運送異常 | 包裹於配送過程發生問題（例如地址錯誤、物流系統異常等） |
| 不需出貨 | 該訂單為虛擬商品或服務，不涉及實體配送 |

## 關鍵字匹配邏輯 { #orders-match-mode }

為提升搜尋效能與精確度，系統針對不同欄位採取兩種搜尋模式。若搜尋不到預期結果，請確認輸入內容是否符合該欄位的匹配規則。

=== "精確匹配（完全一致）"

	以下欄位資訊必須與搜尋內容 **完全相同**，方能查得結果：

	- **顧客資訊**：消費者 Email、會員 Email。

	- **物流資訊**：託運單號 (Tracking Number)。

	- **商品資訊**：商品 SKU。

	- **收件資訊**：收件人電話、收件人完整地址。

	- **金流資訊**：金流端訂單編號。

=== "模糊搜尋（部分一致）"

	以下欄位支援輸入 **部分資訊** 進行查找：

	- **訂單資訊**：訂單編號、優惠券序號、訂單來源（如 POS 使用者 ID）。

	- **商品資訊**：商品名稱。

	- **配送資訊**：收件者姓名、配送方式、倉庫貨態（僅限串接峰潮物流者）。
