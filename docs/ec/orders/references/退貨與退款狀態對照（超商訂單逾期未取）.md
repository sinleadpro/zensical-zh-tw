本對照表彙整超商訂單逾期未取時，**配送狀態**、**退貨狀態**、**付款狀態** 的流轉，以及觸發自動退款的綜合條件。

## 功能介紹 { #intro-overdue-cvs-refund-flow }

當超商系統判定訂單逾期未取後，CYBERBIZ 後台會更新三個欄位：

* **配送狀態**（出貨進度）
* **退貨狀態**（商品退回的位置）
* **付款狀態**（金流退款進度）

這三個狀態的更新邏輯依「訂單是否整筆出貨」、「付款方式是否支援自動退款」、「商家是否安裝相關加值功能」而有所不同。

## 狀態流轉對照 { #reference-overdue-cvs-refund-flow-status-transition }

### 整筆訂單以單一超商物流出貨 { #reference-overdue-cvs-refund-flow-status-transition-whole }

| 階段 | 配送狀態 | 退貨狀態 | 付款狀態 |
| :-- | :-- | :-- | :-- |
| 出貨完成 | 已出貨 | 不需退貨 | 已收到款項 |
| 顧客逾期未取（系統判定） | 逾期未取 | 轉運中心[^1] | 已收到款項 → 待退款 |
| 自動退款執行完成 | 逾期未取 | 轉運中心 → 已退貨 | 已退款 |
| 商品退回完成 | 逾期未取 | 已退貨 | 已退款 |

[^1]: 若商家有安裝「超商逾期未取不自動退款」加值功能，退貨狀態會改為「原寄件門市」而非「轉運中心」，等待商家確認商品狀態。

---

### 部分出貨（第一筆超商 + 第二筆其他物流） { #reference-overdue-cvs-refund-flow-partial }

| 階段 | 第一筆配送狀態 | 第二筆配送狀態 | 退貨狀態 | 付款狀態 |
| :-- | :-- | :-- | :-- | :-- |
| 兩筆出貨完成 | 已出貨 | 已出貨 | 不需退貨 | 已收到款項 |
| 第一筆超商逾期未取 | 逾期未取 | 已出貨（不變） | 原寄件門市 | 已收到款項（**不會自動轉為待退款**） |
| 商家確認後手動退款 | 逾期未取 | 已出貨 | 已退貨 | 已退款 |

!!! warning "注意"
    部分出貨情境下，即使您的店家原本符合自動退款條件，超商品項的逾期 **不會** 觸發系統自動退款。需商家評估商品狀態後手動執行人工退款。

---

## 自動退款支援對照 { #reference-overdue-cvs-refund-flow-auto-refund }

### 哪些付款方式支援自動退款 { #reference-overdue-cvs-refund-flow-auto-refund-payment }

各金流商支援自動退刷的範圍與期限差異請見 [自動退刷時效對照表](自動退刷時效對照表.md){ title="自動退刷時效對照表" data-preview }。

對於超商逾期未取（顧客 4–7 日內未取），所有支援自動退刷的金流都遠在退刷期限內，因此 **金流期限通常不是阻礙**。實際是否觸發自動退款取決於下方綜合條件。

---

### 觸發自動退款的綜合條件 { #reference-overdue-cvs-refund-flow-auto-refund-conditions }

訂單必須 **同時** 滿足下列條件，系統才會在逾期判定後自動執行退款：

- [x] **配送狀態：** 由「已出貨」轉為「逾期未取」
- [x] **付款狀態：** 為「已收到款項」
- [x] **付款方式：** 屬於 [自動退刷時效對照表](自動退刷時效對照表.md){ title="自動退刷時效對照表" data-preview } 列出的金流（且仍在退刷期限內）
- [x] **整筆訂單以同一超商物流出貨：** 非部分出貨
- [x] **未安裝「超商逾期未取不自動退款」加值功能**
- [x] **商家方案支援自動退款金流**

任一條件不符即進入「人工退款」流程，由商家在訂單詳情頁填寫顧客退款資訊。

## 退貨狀態語意對照 { #reference-overdue-cvs-refund-flow-return-status }

| 退貨狀態 | 說明 |
| :-- | :-- |
| 不需退貨 | 預設值，訂單未進入退貨流程 |
| 轉運中心 | 商品已從原寄件門市送往超商物流總倉，逾期未取自動退款的預設值 |
| 原寄件門市 | 商品仍在原寄件門市等待商家領回。安裝「超商逾期未取不自動退款」加值功能時的預設值 |
| 退貨中 | 商品在退回過程中 |
| 已退貨 | 商品已退回商家且退貨流程完成 |
| 運送異常 | 退貨運送過程發生異常 |
