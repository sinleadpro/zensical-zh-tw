## 訂單搜尋與篩選說明

**「訂單搜尋與篩選」** 功能主要集中在後台的訂單管理頁面，旨在協助商家從大量訂單中快速找出目標，並透過自定義檢視來提升管理效率。

## 進入訂單管理 { #orders-path }

登入 CYBERBIZ 管理後台，點選左側選單 **「訂單」** > **「所有訂單」**。

![進入所有訂單](../../assets/images/EC-所有訂單-路徑.png)

## 關鍵字搜尋 { #orders-search }

搜尋欄位支援多種關鍵字，但需注意搜尋的 [匹配規則][orders-match-mode]{ data-preview }：

=== "新版訂單列表"

    1. **輸入關鍵字**：於搜尋框輸入搜尋條件（如訂單編號、姓名或手機），並按下 ++enter++。
    2. **檢視結果**：符合條件的訂單將即時顯示於列表中。
    3. **調整分頁筆數**：透過列表下方的下拉選單，可切換每頁顯示筆數（10 / 25 / 50 / 100 / 200 筆）。

    ![訂單列表-搜尋](../../assets/images/EC-所有訂單-搜尋操作.png)

=== "舊版訂單列表"

    1. **輸入關鍵字**：於搜尋框輸入搜尋條件，並按下 ++enter++。
    2. **檢視結果**：符合條件的訂單將顯示於下方列表中。
    3. **調整分頁筆數**：透過列表 **右上角** 的下拉選單，可調整單頁顯示訂單數量。

    ![訂單列表-搜尋-舊版](../../assets/images/EC-所有訂單-搜尋操作-舊版.png)

## 條件篩選 { #orders-filter }

商家可疊加多個條件，並可搭配 [關鍵字搜尋][orders-search]{ data-preview } 再縮小範圍。

### 篩選條件組合邏輯 { #orders-filter-logic }                                                                
                 
| 情境 | 邏輯 | 範例 |
|---|---|---|
| **同一篩選器內選多個值** | 任一條件符合即可 | 訂單狀態勾選 `進行中` + `已結案` > <br> 兩種狀態的訂單都會出現 |
| **不同篩選器疊加** | 所有條件皆需成立 | 付款狀態 = `已收到款項` + 配送狀態 = `未出貨` > <br> 只列出已收款且尚未出貨的訂單 |
| **關鍵字搜尋 + 篩選條件** | 先縮小範圍再比對關鍵字 | 篩選 `未出貨` 後再搜尋 `0912` > <br> 只會在未出貨訂單中比對手機號碼 |
                  
!!! info "邏輯為固定行為"                                                                                  
    系統不提供「全部符合 / 任一符合」切換鈕。
    舊版每個下拉只能選單一值，因此「同篩選器多選 OR」僅適用於新版。 

---

### 操作方式

=== "新版訂單列表"

    篩選條件以 **可疊加的篩選條件** 呈現，透過 **「+新增篩選條件」** 加入。   

    1.  **新增篩選條件**：點選列表上方 **「+新增篩選條件」**，從下拉中挑選欲 [套用的欄位][orders-v2-filter]{ data-preview }。已新增的條件將以標籤形式顯示。

        ![訂單列表-篩選條件標籤](../../assets/images/EC-所有訂單-已新增篩選條件.png)

    2.  **設定篩選值**：點擊篩選條件標籤展開下拉選單，可選擇多個值。同一篩選條件內的多選結果為「任一符合即可顯示」。

        ![訂單列表-設定篩選條件值](../../assets/images/EC-所有訂單-篩選條件值設定.png)

    3.  **設定時間區間**：**訂單成立日期** 與 **訂單配送日期** 支援兩種模式：                              
      
        - **介於**：指定明確的起訖日期                                                                     
        - **過去**：指定「過去 N 天」的滾動式區間（適合存成檢視群組）
                                                                                    
        ![訂單列表-篩選時區-介於](../../assets/images/EC-所有訂單-篩選時區-介於.png){ .screenshot }    
                                                                                                                
    4.  **清除條件**：移除單一條件 > 點條件標籤旁的 ✕；一次清空所有條件 > 點 **「清除全部」**。              
    5.  **儲存為訂單檢視群組**：篩選設定完成後，點 **「儲存」** 開啟對話框：                               
                                                                                                          
        - **建立新檢視群組**：命名後加入左側群組列表，可一鍵切換                                           
        - **更新檢視群組**：覆寫目前所在的群組設定


    ??? plan "篩選器的開通條件"
          看不到下列篩選器代表方案尚未開通對應功能；於 **「+新增篩選條件」** 中也不會出現：                  
                                                                                                             
          | 篩選器 | 需要開通 |
          | :-- | :-- |                                                                                      
          | 線上 / 線下門市（依 POS 門市篩選） | POS 系統 |
          | 配送方式（含店取選項） | 門市取貨 |                                                              
          | 商品通路 | 商品通路設定 |                                                                        
          | 配送溫層 | 商品綁定溫層 |                                                                        
          | 退貨狀態（部分選項） | 物流系統 |                                                                
          | LINE 團購 | LINE 團購 |                                                                          
          | 商家來源 / 收款方 | 企業版 / PLUS 版 方案，或未啟用「自有金流」 |

## 進階功能：新版訂單列表優化

CYBERBIZ 推出的新版列表提供更強大的自定義工具：

### 訂單檢視群組 { #order-groups-tabs }

透過「檢視群組」功能，商家可將常用的 [篩選條件](references/訂單篩選條件欄位定義.md){ data-preview }  （如：指定物流、付款或配送狀態）組合儲存儲存為自定義頁籤，實現一鍵切換，提升批次處理訂單的效率。

**操作步驟**

1. **設定篩選條件：** 點擊 **新增篩選條件**，根據訂單、付款、配送或退貨狀態勾選所需條件。
2. **儲存群組：** 設定完成後，點擊 **儲存** 並命名該群組（如：「需出貨」）。系統將於頁面頂端建立對應的頁籤。
3. **管理群組：** 點擊頁籤旁的 :lucide-chevron-down: 圖示，可針對現有群組進行「編輯名稱」或「刪除」。

??? example "常用檢視群組範例"

    | 檢視群組名稱 | 建議篩選條件設定 |
    | :--- | :--- |
    | **需出貨訂單** | **配送狀態**：`未出貨`、`準備出貨`<br>**付款狀態**：`貨到付款`、`已收到款項` |
    | **指定物流訂單** | **配送方式**：勾選指定物流商（如：黑貓、全家） |
    | **退貨處理中** | **訂單狀態**：`進行中`<br>**退貨狀態**：`退貨申請`、`退貨中`、`退貨審查` |

![](../../assets/images/ec-所有訂單-檢視群組.gif)

### 編輯列表欄位 { #orders-edit-columns }

- 點選 **編輯欄位**，自由勾選欲顯示的 [訂單列表欄位項目](references/訂單列表欄位參考表.md){ data-preview }（如：會員、商品詳情、訂單標籤、成立日期、付款方式、配送日期等）。
- 透過拖曳 :lucide-grip-vertical: 圖示調整先後順序，建立符合使用習慣的清單。

??? tip "優化定期購追蹤" 
	若經營定期購業務，建議開啟 `母訂單編號` 與 `配送期數` 欄位，以便在列表中快速掌握子訂單的計畫歸屬。

---

商家可以彈性選擇列表要顯示的資訊欄位並調整排序。

*   **可選欄位**：包含會員資訊、商品詳情、訂單標籤、付款/配送方式、發票狀態、母訂單編號等。
*   **排序功能**：點選 **「編輯欄位」** 後，可拖曳 `⋮⋮` 圖示來調整欄位出現的先後順序。

### 訂單標籤註記

新增 **「訂單標籤」** 功能，商家可以在操作訂單時手動貼上標籤進行分類註記，並搭配檢視群組使用。

## 其他特定搜尋路徑

*   **定期定額訂單**：需至「訂單」>「定期定額訂單」中搜尋，支援定期單號、收貨資訊或備註搜尋。
*   **POS 訂單**：在 POS 前台可透過輸入訂單號碼、掃描發票或進階篩選（如成立時間、認單時間）來查找。
*   **會員專屬搜尋**：進入特定會員頁面後，可點選其「訂單」分頁，查看該會員在 EC 與 POS 的所有有效消費紀錄。

訂單列表 — 如何搜尋與篩選商品（使用指南）

> 適用頁面：
> - 新版訂單列表：`/admin/orders_v2`
> - 舊版訂單列表：`/admin/orders`
>
> 對象：CYBERBIZ 後台商家管理者

---

## 1. 功能介紹

訂單列表是後台中使用頻率最高的頁面之一。當訂單累積到上百、上千筆時，您需要透過「**搜尋**」與「**篩選**」快速找到特定訂單，例如：

- 某張團購訂單裡到底買了哪幾項商品？
- 哪些訂單包含「冷凍」溫層的商品，需要優先安排冷鏈出貨？
- 客戶說只買了某個 SKU，但找不到訂單編號，能不能用商品反查？
- 我想一次查所有透過「LINE 團購」管道下的訂單，方便對帳。

本指南聚焦於**以「商品」為角度切入**的搜尋與篩選技巧，並比較新版 (`orders_v2`) 與舊版 (`orders`) 的操作差異。

> 兩個版本的訂單列表並存於系統中，您可以從後台切換使用。新版以全新介面與更豐富的篩選器為主，舊版仍保留作為過渡期使用。

## 3. 規則說明（版本與功能限制）

並不是每位商家都看得到所有篩選器。以下篩選功能受到「**店舖版本**」或「**加值功能**」影響，看不到的話表示您目前的方案尚未開通。

### 3.1 隨版本內建的功能

| 篩選器 | 中文名稱 | 內建版本 |
| :-- | :-- | :-- |
| `manual_order` | 手動訂單 | 幾乎所有版本（試用版除外） |
| `product_bind_temperature` | 商品綁溫層 | 專業版、專業 plus、進階版、進階 plus、高手版、高手 plus、尊爵版、企業版 |
| `branch_store_setting` | 門市設定 | 專業 plus、進階 plus、高手版、高手 plus、POS 獨賣、企業版 |
| `product_bind_channel` | 商品綁通路 | 高手版、高手 plus、進階 plus、專業 plus、尊爵版、企業版 |

### 3.2 需另外加購的功能

下列功能未隨任何版本內建，需另行開通或加購：

- **進階訂單搜尋**（`advance_order_search`）— 啟用 Elasticsearch 搜尋引擎、支援「完全符合」精確搜尋
- **POS 門市管理**（`pos`）— 啟用後才會出現「門市來源」篩選
- **LINE 團購**（`line_group_buy`）— 啟用後才會出現「LINE 團購訂單」類型篩選
- **門市取貨**（`store_pickup`）— 出現於「配送方式」篩選
- **多倉庫**（`partial_warehouse`）— 出現於舊版的「倉庫類型」篩選
- **訂單標籤 API**（`api_order_tags`）— 新版「訂單標籤」篩選

> 若您不確定目前店家屬於哪個版本，可至「**店家設定 ▸ 版本與訂閱**」查看，或聯繫您的客戶成功經理。

## 4. 操作步驟（依使用情境）

### 情境 A：用「商品名稱」找訂單

> 例：客戶來電說購買的是「焙煎黑豆茶」，但忘了訂單編號。

#### 新版 `/admin/orders_v2`

1. 進入「訂單管理 ▸ 訂單」（新版）。
2. 在頁面上方搜尋框旁的下拉選單，選擇 **「商品名稱」**。
3. 在搜尋框輸入 `焙煎黑豆茶`，按 Enter 或點放大鏡圖示。
4. 系統列出所有「至少包含一個品項名稱與此關鍵字相符」的訂單。
5. （選用）若您想精確比對完整商品名稱，勾選 **「完全符合」**（需開通「進階訂單搜尋」）。

#### 舊版 `/admin/orders`

1. 進入「訂單管理 ▸ 訂單」（舊版）。
2. 將搜尋框旁的「搜尋類型」切換為 **「商品名稱」**。
3. 輸入關鍵字 > 送出。
4. （選用）勾選 **「完全符合」**（同樣需「進階訂單搜尋」）。

> **小提示**：若一張訂單裡有兩項商品都符合關鍵字，該訂單只會在列表中出現一次。

### 情境 B：用「商品 SKU」找訂單

> 例：發現某 SKU 的庫存對不上，想反查所有銷售出去的訂單。

兩版操作相同：

1. 開啟訂單列表。
2. 將搜尋類型切到 **「商品 SKU」**。
3. 輸入完整 SKU（例如 `BLACKTEA-001`）> 送出。
4. 預設為部分符合；若要精準比對，請勾選「**完全符合**」。

### 情境 C：用「商品通路」篩選訂單（**僅新版**）

> 適用情境：商家同時上架多個通路（自有官網、Yahoo 拍賣、PChome 商店街等），想看「PChome 通路」近期賣了哪些訂單。

> 前置：本店家版本須包含 `product_bind_channel`（高手版／高手 plus／進階 plus／專業 plus／尊爵版／企業版），且已於商品設定指定通路。

1. 進入新版訂單列表。
2. 點選頁面上方的 **「篩選」** 按鈕，展開篩選面板。
3. 找到 **「商品通路」** 區塊。
4. 勾選您要查看的通路（可多選），例如 `PChome`。
5. 套用篩選 > 列表只剩「至少包含一項該通路商品」的訂單。
6. 若想看「沒有設定通路」的訂單，可勾選 **「無通路」** 選項。

---

### 情境 D：用「配送溫層」篩選訂單（**僅新版 + 加值**）

> 適用情境：今天要安排冷鏈車次，先把所有需要冷凍／冷藏出貨的訂單抓出來。

> 前置：店家須安裝「商品綁溫層」`product_bind_temperature`（含於專業版以上）。

1. 進入新版訂單列表 > 開啟篩選面板。
2. 在 **「配送溫層」** 中勾選您需要的溫層：
   - 常溫
   - 冷藏
   - 冷凍
3. 套用後，只列出「**任一品項**屬於該溫層」的訂單。
4. 可結合「配送日期」進一步縮小到「明天要出貨的冷凍訂單」。

> **注意**：一張訂單若同時含常溫＋冷凍商品，勾選任一溫層都會把它列出來。實務上會建議搭配「**訂單標籤**」做出貨流程分組。

### 情境 E：篩選 LINE 團購訂單

> 前置：店家須安裝「LINE 團購」`line_group_buy`。

1. 開啟訂單列表（新版或舊版皆可）。
2. 在篩選面板中找到 **「訂單類型」**（舊版）或 **「LINE 團購」**（新版）。
3. 選擇 **「LINE 團購訂單」**。
4. 列表只顯示透過 LINE 團購流程進入的訂單，方便獨立對帳與出貨。

### 情境 F：搜尋 + 篩選組合（最常見的工作流）

> 例：找出「最近 7 天、付款狀態為已付款、含商品『中秋禮盒』」的所有訂單。

1. 開啟新版訂單列表。
2. 設定 **訂單成立日期** > 「過去 7 天」。
3. **付款狀態** > 「已付款」。
4. 切換搜尋類型為 **「商品名稱」**，輸入 `中秋禮盒`。
5. 套用篩選並送出搜尋。
6. （建議）將此查詢結果用「**匯出**」功能下載 Excel／CSV 留存。

> **小提示**：搜尋條件與篩選條件會「**同時成立（AND）**」。日期區間越大，搜尋越慢；建議先縮小日期再下關鍵字。

## 5. 後續操作

找到目標訂單後，常見的接續動作：

1. **批次出貨**：勾選多筆訂單 > 上方「批次操作」> 選擇對應物流（黑貓、全家、7-11、宅配通…等）。
2. **列印揀貨單／出貨單／發票**：批次操作 > 列印。
3. **修改訂單狀態**：個別點入訂單編輯，或透過批次操作關閉／開啟訂單（需「手動訂單」）。
4. **匯出**：點擊頁面右上的「匯出」可下載目前查詢結果為 Excel／CSV 檔，便於財會或客服進一步處理。
5. **訂單標籤**（新版）：對篩選結果一次性貼上標籤，例如 `2026年中秋出貨`，下次直接靠標籤調出。

## 6. 常見問題（FAQ）

**Q1. 為什麼我的訂單列表沒有「商品通路」或「配送溫層」這個篩選器？**
A：這些篩選器需要對應的功能（`product_bind_channel`、`product_bind_temperature`）已啟用。請至「店家設定 ▸ 版本與訂閱」確認版本，或聯繫客戶成功經理升級／加購。

**Q2. 「完全符合」勾跟不勾差在哪？**
A：
- **不勾（預設，模糊搜尋）**：只要商品名稱「**包含**」關鍵字就會被找到。例如搜尋「黑豆茶」，會找到「焙煎黑豆茶」、「無糖黑豆茶 500ml」。
- **勾選（完全符合）**：必須**完全相同**才會被找到，適合您已經知道完整名稱／完整 SKU 時使用。
- 「完全符合」需開通「**進階訂單搜尋**」加值功能。

**Q3. 我用商品名稱搜尋，但確實有賣這個商品卻找不到訂單？**
A：可能原因：
1. 訂單成立時，商品名稱與您現在輸入的不一致（例如後來改名）。系統紀錄的是**下單當下**的品項名稱。
2. 該商品被列為「贈品」或「加購」，且設定不納入搜尋。
3. 已開啟「完全符合」但關鍵字非完整名稱 > 取消勾選再試一次。
4. 訂單已被刪除（非關閉），列表只顯示未刪除的訂單。

**Q4. 新版跟舊版結果會一樣嗎？**
A：基本上一致，但新版的搜尋條件更豐富（如「訂單標籤」「商品通路」「配送溫層」），且支援更彈性的日期區間（例如「過去 30 天」）。同樣的關鍵字在兩版查詢結果應相同。

**Q5. 我想找「同時包含 A 商品和 B 商品」的訂單？**
A：目前搜尋框一次只接受一組關鍵字。建議：
1. 先以 A 商品搜尋並匯出。
2. 再以 B 商品搜尋並匯出。
3. 在試算表交叉比對訂單編號。
4. 若您頻繁有此需求，可考慮使用「**訂單標籤**」搭配自動化規則。

**Q6. 篩選結果太多／太慢怎麼辦？**
A：
1. 先用日期區間（例如最近 7 天）大幅縮小範圍。
2. 加上付款／配送狀態，例如只看「已付款 + 未出貨」。
3. 若仍然很慢，建議開通「**進階訂單搜尋**」（Elasticsearch），可顯著提升大資料量下的查詢速度。

**Q7. 篩選條件設定後可以保存嗎？**
A：新版透過 URL 保留篩選狀態，您可以將「過濾後的網址」加入瀏覽器書籤，下次點開即還原相同條件。舊版亦同。

## 參考資料

### 關鍵字匹配邏輯 { #orders-match-mode }

為提升搜尋效能與精確度，系統針對不同欄位採取兩種搜尋模式。若搜尋不到預期結果，請確認輸入內容是否符合該欄位的匹配規則。

=== "精確匹配（完全一致）"

	以下欄位資訊必須與搜尋內容 **完全相同**，方能查得結果：
	
	- **顧客資訊**：消費者 Email、會員 Email。
	    
	- **物流資訊**：託運單號 (Tracking Number)。
	    
	- **商品資訊**：商品 SKU。
	    
	- **收件資訊**：收件人電話、收件人完整地址。
	    
	- **金流資訊**：金流端訂單編號。
	    

=== "模糊搜尋（部分一致）"

	以下欄位支援輸入 **部分資訊** 進行查找：
	
	- **訂單資訊**：訂單編號、優惠券序號、訂單來源（如 POS 使用者 ID）。
	    
	- **商品資訊**：商品名稱。
	    
	- **配送資訊**：收件者姓名、配送方式、倉庫貨態（僅限串接峰潮物流者）。
	    
	- **帳單資訊**：城市、區域、地址。（需分開搜尋，不可直接搜整串地址）

### 訂單篩選條件對照表 { #orders-filter }

!!! info "可供選擇的篩選欄位與選項值，將根據您的 **使用方案** 或 **後台功能設定** 而動態調整。"

以下為系統內所有可用的篩選條件說明：

=== "狀態類"

	用於追蹤訂單生命週期中的關鍵節點。
	
	- **訂單狀態：** 訂單的總體進度（如：進行中、已完成、已取消）。
	    
	- **付款狀態：** 資金對帳情況（如：未付款、已收到款項、退款中）。
	    
	- **配送狀態：** 物流處理進度（如：未出貨、已出貨、已到達）。
	    
	- **退貨狀態：** 售後流程進度（如：退貨申請、退貨審查、已退貨）。
    

=== "通路與標籤"

	用於區分訂單來源與自定義分類。
	
	- **商品通路：** 區分訂單來源（如：官網、門市、第三方平台）。
	    
	- **LINE：** 篩選透過 LINE 相關功能（如：LINE 購物、LINE OA）產生的訂單。
	    
	- **收款方：** 若商家有多個收款帳號，可用於區分金流歸屬。
	    
	- **訂單標籤：** 根據商家自行貼上的標籤進行分類（如：VIP 客戶、急件）。
    

=== "日期區間"

	- **訂單成立日期：** 消費者下單的原始時間。
	    
	- **訂單配送日期：** 系統紀錄或商家填寫的預計/實際出貨時間。


### 7.4 與商品相關的篩選器版本對照

### 7.5 限制條件

- 大量資料下，建議搭配「日期區間」使用以避免逾時。
- 「完全符合」依賴 Elasticsearch；若未開通「進階訂單搜尋」，僅支援模糊比對。
- 已刪除的訂單不會出現在搜尋／篩選結果中。
