# 出貨狀態物流提示文字說明

串接物流訂單出貨後，系統如何依物流貨態自動顯示不同的「已出貨」提示文字，以及其顯示位置與通知觸發時機。
{ .subtitle }

[:lucide-toggle-right:{ title="適用功能" }](../../resources/conventions#適用功能) | 拖拉版型
{ .doc-badge }

![](../../assets/images/ec-會員-訂單查詢頁-物流提示文字.png){ .hero-page }

在 CYBERBIZ 系統中，**「已出貨狀態物流提示文字」** 功能旨在提供商家與顧客更精確的訂單配送資訊，讓雙方能即時掌握包裹目前的處理進度。

以下為該功能的詳細說明與教學：

## 功能定義與支援範圍

- **觸發機制**：當商家針對「**串接物流**」訂單操作出貨，點選「下載託運單並改為已出貨」後，系統會根據物流商回傳的貨態，在「已出貨」欄位旁加上補充文字。

- **支援對象**：

	- **僅限串接物流訂單**：必須透過系統下載託運單。若使用「自訂物流」出貨，配送狀態僅會停留在「已出貨」，不會顯示補充文字。

	- **僅限拖拉版型使用**：此功能不支援預設版型。

	- **適用訂單類型**：包含一般訂單與定期定額訂單。

## 相關注意事項

- **託運單失效**：若託運單產出後  **14 日內** 未寄件，該單號將失效。此時訂單列表的配送狀態會保留在「已出貨」，但明細頁內的提示文字會持續停留在「**已出貨 (待物流收件)**」。

- **物流聯繫**：下載託運單後（狀態顯示為待物流收件），商家需依各物流商規範聯繫司機，待司機收件並更新系統後，狀態才會轉為「配送中」。

## 物流提示文字種類與定義

系統共有兩種類型的補充文字，分別代表不同的物流階段：

|提示文字|對應情境與物流資訊|
|---|---|
|**已出貨 (待物流收件)**|商家已於後台產出託運單，但物流商 **尚未收件**，或已收件但 **尚未回傳貨態**。此時包裹尚未正式進入配送階段。|
|**已出貨 (配送中)**|商家已完成出貨作業並將包裹交給物流商，且 **物流商已回傳物流貨態**。代表包裹已正式進入配送階段。|

## 文字顯示位置

!!! info "物流提示文字 *不會* 顯示在後台的「訂單列表頁」或「篩選器」中。"

物流提示文字會出現在以下特定介面，方便商家與顧客查看：

### 後台管理中心

- **訂單明細頁**（路徑：「訂單」>「所有訂單」> 點擊指定訂單編號）。

![](../../assets/images/ec-訂單明細-物流狀態提示文字.png)

### 前台會員中心

- **訂單查詢頁** （路徑：**網站右上角 :lucide-user: 圖示 > 訂單查詢**）。

	![](../../assets/images/ec-會員-訂單查詢頁-物流提示文字.png)

- **訂單明細頁**（路徑：**網站右上角 :lucide-user: 圖示 > 訂單查詢 > 訂單明細**）。

	![](../../assets/images/ec-會員-訂單明細頁-物流提示文字.png)


## 自動化出貨通知發送時機

若商家有開啟 Email、簡訊或 LINE OA 的出貨通知樣板，系統發送訊息的時間點如下：

- **發送條件**：當訂單配送狀態更新為「**已出貨 (配送中)**」時，系統才會正式發送通知給會員。

- **目的**：確保消費者收到通知時，包裹已確實由物流商承運，避免因尚未收件而產生的查詢困擾。

## 常見問題

??? quote "為什麼我的訂單已經交給司機了，後台還是顯示「已出貨 (待物流收件)」"
	這通常是因為物流商尚未完成刷單入庫或系統回傳資料有時間差（延遲）。請耐心等候 4–8 小時，待物流端更新貨態後，系統文字將自動轉為「已出貨 (配送中)」。

??? quote "如果我誤點了「下載託運單」，可以手動將狀態改回「未出貨」嗎" 
	一旦點選「下載託運單並改為已出貨」，系統即視為執行出貨作業。若需取消，必須先進行「取消託運單」動作，訂單狀態才會回歸。請注意，若單號已產出超過 14 日，則無法再進行後續物流追蹤。

??? quote "為什麼有些訂單後台只有顯示「已出貨」，旁邊沒有括號提示文字" 
	請確認該訂單是否為「串接物流」。若您是使用「自訂物流」（如：商家自送、非系統串接之貨運），系統無法接收第三方物流即時貨態，因此僅會顯示基礎的「已出貨」狀態。

??? quote "消費者反應沒收到出貨簡訊，但我看訂單已經是「已出貨 (待物流收件)」" 
	系統的出貨通知（Email/簡訊/LINE）是觸發於「配送中」階段。若狀態停留在「待物流收件」，表示物流端尚未正式收錄該包裹，通知將會在狀態變更為「配送中」時才自動發出。
