# Pandago 配送異常規範

針對快速到貨配送過程中發生的各類異常情境（如地址錯誤、聯繫不到客戶、配送延遲等），提供責任歸屬判定與操作指引。
{ .subtitle }

[:lucide-tag:{ title="適用方案" }](../../resources/conventions#適用方案) | 所有PLUS / 企業
{ .doc-badge }




## 使用須知

- **適用情境**：本文件所列之異常情境，皆為 **消費者未收到貨** 的情況。
- **配送地址不可修改**：快速到貨訂單因與外送系統即時串接且受區域限制，**訂單成立後無法修改地址**。
- **取消判定**：若送貨狀態顯示為 **訂單已取消**，且退貨狀態為 **不需退貨**，則判定為外送平台端主動取消訂單。
- **運費退還機制**：第一段運費會由系統事先收取。若後續判定為平台責任而不需支付運費，外送平台將於下個月帳期退還給 CYBERBIZ，由 CYBERBIZ 再行退還給商家（反映於企業版對帳單）。
- **逆物流運費成本**：若因消費者因素導致商品退回門市（逆物流），商家通常需負擔雙倍運費（原配送運費 + 逆物流運費）。
- **賠償申請上限**：若因外送平台責任導致商品毀損或遺失，賠償上限通常為單筆 **NT$5,000**（依各平台最終判定為準）。
- **每月彙報**：CYBERBIZ 會於每月對帳時彙整異常原因，並調整對帳單中的收費項目。

!!! warning "取消原因與責任歸屬規範"
    關於 **取消原因與責任歸屬** 之詳細判定標準，若未來有所異動，請以 [Pandago 官方網站](https://vendor.foodpanda.com.tw/pandago) 最新公告為準。



## 異常責任歸屬與運費判定矩陣

當發生「消費者未收到貨」的情境時，請依據下表判斷運費負擔與退款建議：

| 異常情境 | 外送平台收運費 | 商品退回門市 | 逆物流費負擔 | 商家對顧客退款建議 |
| :--- | :---: | :---: | :---: | :--- |
| **無可媒合之司機** | 否 | 沒有配送 | 無 | **退全額** (含運費) |
| **商家關店/休店仍呼叫司機取貨** | 是 | 沒有配送 | 無 | **退全額** (含運費) |
| **超出區域配送營業時間** | 否 | 沒有配送 | 無 | **退全額** (含運費) |
| **天氣/災害因素，司機取件後無法配送** | 否 | 是 | 免負擔 | **退全額** (含運費) |
| **配送延遲 (>30分)** | 否 | 是 | 商家平台各半 | **退全額** (含運費) |
| **司機交通意外/失聯** | 否 | 否 (有補償) | 無 | **退全額** (含運費) |
| **司機收件後，發現超出運送範圍** | 否 | 是(再加補償) | 無 | **退全額** (含運費) |
| **收件地址有誤** | 是 | 是 | **商家負擔** | 退商品金額，**不退運費** |
| **聯繫不到客戶 (>10分)** | 是 | 是 | **商家負擔** | 退商品金額，**不退運費** |
| **送件錯誤** | 是 | 是 | **商家負擔** |**退全額** (含運費) |
| **商品溢漏/損壞** | 否 | 否 (有補償) | 無 | **退全額** (含運費) |
| **系統技術問題** | 否 | 是(再加補償) | 無 | **退全額** (含運費) |


## 操作流程

### 任務一：處理「地址有誤」或「聯繫不到人」

這是最常見的異常，請依以下流程處理：

1. **確認異常**：在訂單詳情頁查看 **物流訊息**，若顯示 **聯繫不到收件人** 或 **送件錯誤**。
2. **主動聯絡**：門市人員應嘗試撥打消費者電話確認地址或提醒取貨。
3. **引導結案**：若司機已將貨送回門市，請將訂單標記為 **已退貨**。


### 任務二：處理「商品損壞」或「配送延遲」

1. **蒐證紀錄**：若消費者反映商品損壞，請其提供照片。
2. **辦理全額退款**：由於此類責任歸屬外送平台，請為消費者辦理 **整單退款**（含運費）。
3. **向平台回報**：記錄該筆託運單號與損壞狀況。

## 常見問題

??? quote "快速到貨訂單的地址真的完全不能改嗎？"
    是的。後台不支援直接修改地址功能。若地址微調（如 3 樓改 5 樓）且門市願意，可在「管理員備註」記錄並由門市直接致電司機溝通；但若地址變更跨區，則必須取消訂單重新下單。

??? quote "什麼是「逆物流運費」？為什麼我要付兩次錢？"
    快速到貨使用的是即時外送服務，司機將商品從門市送往客戶端算一趟費用；若無法送達需「送回原門市」，司機同樣付出了時間與勞力，因此外送平台會收取回程費用（即逆物流費）。

??? quote "我如何知道我會得到「訂單補償」？"
    若發生司機失聯、灑餐或遭遇事故，外送平台會回傳特定的狀態碼至後台。您可以在訂單詳情的「操作記錄」或「物流訊息」中看到相關說明。補償金額將會反映在您的月結對帳單中，而非即時撥款。
