# 了解分潤功能規格
了解分潤功能的運作機制、適用對象與情境，協助您選擇最適合的行銷分潤方案。
{ .subtitle }

[:lucide-layers:{ title="適用產品" }](../../resources/conventions#適用產品) | EC / POS
[:lucide-lock:{ title="適用方案" }](../../resources/conventions#適用方案) | 進階 / 高手 / 所有PLUS / 企業
{ .doc-badge }


分潤是促進內、外部合作夥伴共同提升營收的有效工具。透過網紅、部落客引流或內部員工推廣，在訂單成交後自動計算並分配利潤。CYBERBIZ 提供涵蓋線上官網與線下門市的全方位分潤解決方案，協助商家解決虛實整合過程中的業績歸屬與獎勵分配問題。

!!! tip "應用情境"
	- **KOL/網紅行銷**：與外部合作夥伴簽署分潤協議，透過專屬連結或推薦碼追蹤導購成效。
	- **員工激勵制度**：鼓勵門市人員引導顧客註冊會員或推薦購物，將線上業績歸屬於推廣員工。
	- **舊客帶新客**：激勵現有會員分享推薦碼給好友，達成「雙向回饋」的口碑傳播效果。
	- **門市業績歸屬**：在 POS 結帳或到店取件時，精準紀錄協助結帳或提供服務的員工業績。

---

## 分潤機制總覽

根據 **業績歸屬邏輯** 與 **發生場景** 的不同，系統提供以下四種核心分潤模式：

### 推薦人分潤

最靈活且廣泛使用的模式。推薦人（如網紅、員工、會員）提供專屬連結或推薦碼給消費者。

- **線上行為**：消費者點擊專屬連結或在結帳頁輸入推薦碼，系統自動追蹤該筆訂單。
- **線下行為**：消費者在門市結帳時提供推薦碼，由 POS 人員輸入以紀錄分潤。
- **分潤類型**：支援 **整筆訂單分潤** 或 **指定商品分潤**。

!!! tip "如何選擇分潤方式？"
    - **整筆訂單分潤**：適合與 KOL 或外部通路合作，依據 **訂單總額** 快速結算，設定最便利。
    - **指定商品分潤**：適合商品利潤差異大或包含代理件的情境，可 **逐項精確** 設定比例，實現精細化的獲利管理。

### 註冊人分潤

強調 **永久綁定** 關係。當消費者在註冊時填入推薦人的代碼，該消費者便與推薦人永久綁定。

- **運作邏輯**：綁定成功後，該消費者未來不論線上或線下消費，系統皆會自動計算分潤給註冊人。
- **適用場景**：適合門市店員招募長期會員，或長期合作的經銷夥伴。

### 結帳人分潤 

- **運作邏輯**：在門市 POS 結帳時，由操作人員手動選擇 **結帳人**，將該筆訂單的獎勵歸屬於該人員。
- **適用場景**：門市內部櫃位輪班、共同服務情境下的業績分配。

### 取件分潤

- **運作邏輯**：針對 **線上訂購、線下取件** 的訂單，當訂單包裹到店、消費者取貨時，由操作人員手動選擇 **點貨人員**，將該筆訂單的獎勵歸屬於該人員。
- **適用場景**：計算門市人員協助取貨服務的勞務獎勵。
- **分潤類型**：支援 **到店分潤** 或 **取貨分潤**。

## 分潤對象與回饋對照

針對不同類型的分潤合作對象，系統預設之分潤機制適用範圍如下表所示：

| 分潤對象 | 推薦人分潤 | 註冊人分潤 | 結帳人分潤 | 取件分潤 |
| ------- | --------- | ---------- | -------- | -------- |
| **第三方 (網紅/團媽)** | ✓ | ✕ | ✕ | ✕ |
| **一般顧客 (會員)** | ✓ | ✕ | ✕ | ✕ |
| **店長/店員 (員工)** | ✓ | ✓ | ✕ | ✕ |
| **實體門市 (POS店)** | ✓ | ✓ | ✓ | ✓ |

系統可針對不同的合作對象，設定對應的獲取方式與成效查詢權限：

| 分潤對象 | 推薦碼取得方式 | 成效查詢方式 | 建議回饋方式 |
| :--- | :--- | :--- | :--- | 
| **第三方 (網紅/團媽)** | 商家提供專屬代碼 | 商家提供專屬報表連結 | 廣告費、勞務報酬 |
| **一般顧客 (會員)** | 前台 會員中心 > 個人資訊  | 前台 會員中心 > 個人資訊 | 紅利點數 |
| **店長/店員 (員工)** | 後台 分潤 > 分潤查詢 | 後台 分潤 > 分潤報表 | 業績獎金 (商家自行發放) |
| **實體門市 (POS店)** | 後台 分潤 > 分潤查詢 | 後台 分潤 > 分潤報表 | 門市公積金/團績獎金 |

!!! warning "POS 系統相容性說明"
    實體門市分潤功能僅限於配合 CYBERBIZ POS 系統時使用。若您目前使用其他系統，建議與我們的顧問聯繫以評估轉換方案。


## 核心運作規則與限制

在規劃分潤方案前，請務必了解以下系統運作邏輯：

### 1. 獎金結算時機
- 系統僅在訂單狀態變更為 **已結案** 時，才會正式計算分潤金額並發放紅利回饋。
- 若訂單已結案後發生退貨或取消，系統 **不會** 自動補送或收回已發放的紅利。

### 2. 金流處理原則
- CYBERBIZ 後台僅提供 **分潤報表** 供統計參考，系統 **不經手** 實際的現金發放。
- 商家可根據報表數據，自行透過線下撥款、轉帳或系統紅利點數進行給付。

### 3. 設定效期限制
- 訂單必須在方案的 **生效期間** 內下單，才符合分潤資格。
