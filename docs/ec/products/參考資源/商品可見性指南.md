# 商品可見性指南
設定與管理商品在網站前台的顯示狀態，精準控制商品的曝光與銷售。
{ .page-subtitle } 

---

## 說明概述

=== ":material-information-outline: 注意事項"
	- [ ] **設定立即生效**：除搜尋引擎索引外，所有商品可見性設定變更會立即生效於網站前台。
	- [ ] **搜尋引擎差異**：設定「排除搜尋」僅影響網站內部搜尋，外部搜尋引擎（如 Google）需等待其重新爬取網站後才會更新結果。
	- [ ] **連結存取**：即使商品被設定為「排除搜尋」，若顧客持有直接商品連結，仍可透過連結進入商品頁面並購買（需商品為「公開」且在「上架狀態」）。

=== ":material-lightbulb-outline: 應用情境"
	- 避免優惠價曝光：將促銷商品設定為排除搜尋，僅提供「商品連結」給特定顧客。
	- 管理未上架商品：將尚未準備好的商品設為不公開，避免提前曝光。
	- 控制促銷時間：利用上架狀態功能，精準控制商品上架與下架的時間點。

## 前台顯示邏輯

商品的最終可見性，取決於「公開狀態」、「上架狀態」及「排除搜尋」三個功能的綜合設定。了解其邏輯，能幫助您更精準地管理商品曝光。

1. **公開狀態 (必需條件)**

	*   若商品設定為「不公開」，則無論上架狀態或排除搜尋如何設定，商品皆不會顯示於網站前台，且無法透過直接連結購買。
	*   若商品設定為「公開」，則需進一步判斷「上架狀態」與「排除搜尋」。

2.  **上架狀態 (時間控制)**
 
	*   若商品已設為「公開」，則其僅會在設定的「上架時間」與「下架時間」區間內顯示於前台。
	*   若不在排程時間內，商品將無法被看到或購買。

 3. **排除搜尋 (曝光範圍)**
 
	*   若商品已設為「公開」且在「上架時間」內，但同時設定了「排除搜尋」：
	- [x] 商品將不會出現在網站內部搜尋結果、商品分類頁面或首頁推薦中。
	- [x] 商品仍可透過「直接連結」被存取及購買。
	- [x] 外部搜尋引擎（如 Google）不會索引此商品頁面。

> **建議使用策略**  
> - 若商品需要**完全隱藏**，優先使用「不公開」。  
> - 若商品僅需**限制曝光範圍**，使用「排除搜尋」即可。

## 常見問題

??? question "「不公開」與「排除搜尋」有什麼區別？"
    「不公開」的商品在網站前台完全不可見，也無法透過直接連結存取。而「排除搜尋」的商品在網站前台仍可透過直接連結存取，但不會顯示於搜尋結果或分類頁面中。

??? question "設定商品「排除搜尋」後，多久會從 Google 搜尋結果中消失？"
    Google 搜尋結果的更新取決於其爬蟲的頻率。通常需要數天到數週，無法立即生效。建議在重要資訊需要隱藏時，提早設定或優先使用「不公開」功能。

??? question "已設定為「不公開」的商品，顧客可以透過直接連結購買嗎？"
    不可以。被設定為「不公開」的商品，即使有直接連結也無法在前台被存取或購買。此狀態主要用於測試或未準備好的商品。
