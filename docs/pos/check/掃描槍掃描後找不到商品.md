# 掃描槍掃描後找不到商品
解決在使用掃描槍掃描商品條碼時，系統無法辨識或找不到對應商品的問題，包含 EAN-13 與 UPC-A 編碼轉換設定。
{ .subtitle }

[:lucide-tag:{ title="適用方案" }](../../resources/conventions#適用方案) | 進階 PLUS / 高手 PLUS / 企業
{ .doc-badge }

!!! tip "應用情境"
	- **掃描無反應**：掃描商品條碼後，POS 系統沒有帶入商品資訊。
	- **條碼不匹配**：掃描出的數字與商品標籤上的數字不一致（例如少了一位數）。
	- **新設備啟用**：剛更換掃描槍，需要進行初始編碼對齊。


## 使用須知

- **SKU 碼檢查**：請確保已在 CYBERBIZ 管理後台為商品建立對應的 SKU 碼或條碼。
- **編碼差異**：北美常用的 UPC-A（12 碼）與國際常用的 EAN-13（13 碼）在掃描時可能因首位「0」的處理方式不同而導致辨識失敗。
- **設備支援**：本教學適用於 CYBERBIZ 提供之掃描槍。若使用非官方提供設備，請洽詢您的設備供應商。

## 操作流程

### 步驟1：確認後台資料

1. 登入 CYBERBIZ POS 管理後台，前往 **商品 > 所有商品**。
2. 確認該商品的 **SKU 碼** 或 **條碼** 欄位已填入正確的數字編碼。

### 步驟2：檢查條碼編碼格式

若您的商品條碼為 EAN-13 格式（13 位數字），請檢查標籤上的第一位數字是否為 **0**。

![](https://www.cyberbiz.co/support/wp-content/uploads/2019/10/EAN_0-1.png){ .small-image }

### 步驟3：驗證掃描槍讀取結果

1. 在電腦上開啟瀏覽器，點擊搜尋列。
2. 使用掃描槍掃描上方條碼。
3. 觀察輸出的數字：
    - **輸出為 0012345678905**：掃描槍設定正確。
    - **輸出為 012345678905**：代表掃描槍目前使用 UPC-A 編碼讀取，自動省略了首位的區域碼 0。

### 步驟4：修正掃描槍設定

若確認掃描槍遺漏了首位數字 0，請使用您的掃描槍掃描下方設定條碼。聽到 **Bi Bi Bi~** 聲響後，即代表完成「UPC-A 轉換為 EAN-13」的設定。

![](https://www.cyberbiz.co/support/wp-content/uploads/2019/10/image-1.png){ .small-image }



!!! info "EAN-13 與 UPC-A 編碼"
    - **UPC-A**：最早使用的 12 碼編碼，主要用於美國與加拿大。
    - **EAN-13**：國際標準編碼，在 UPC-A 前加上一位區域碼。美國與加拿大的區域碼為 **0**，但在某些掃描槍預設中不會顯示該位數。
    

## 常見問題

??? quote "掃描後出現亂碼怎麼辦？"
    這通常是因為掃描槍的輸入法設定不正確。請確保掃描時電腦或 POS 機的輸入法處於 **英文半形** 狀態。

??? quote "非 CYBERBIZ 提供的掃描槍也可以這樣設定嗎？"
    不同品牌的掃描槍設定條碼不同。若您使用自有設備，請查找該型號的使用說明書，尋找 **UPC-A to EAN-13** 或 **Transmit UPC-A Check Digit** 相關設定。
