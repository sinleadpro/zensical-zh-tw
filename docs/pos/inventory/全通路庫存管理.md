# 全通路庫存管理
本指南將引導您了解 CYBERBIZ POS 與 EC 系統的庫存協作機制，並提供進倉、調倉、出倉及盤點的標準操作流程。
{ .subtitle }

[:lucide-tag:{ title="適用方案" }](../../resources/conventions#適用方案) | 進階 PLUS / 高手 PLUS / 企業
{ .doc-badge }


!!! tip "應用情境"
    - **跨通路庫存同步**：當商家同時經營官網與實體門市，需區分並同步 EC 與 POS 庫存。
    - **門市間貨物調度**：門市 A 缺貨時，向門市 B 申請調貨，並自動完成出進倉流程。
    - **定期庫存盤點**：門市人員定期清點實際庫存，並透過系統校正帳面數量。


## 理解系統分倉與模式

在開始操作前，請務必理解 CYBERBIZ 的庫存架構，這將決定您的日常營運流程。

### 系統分倉結構

CYBERBIZ 系統將 **EC 官網庫存** 與 **POS 門市庫存** 視為獨立倉儲管理：

1. **EC 倉**：管理官網訂單的出貨庫存。
2. **POS 門市倉**：各實體門市擁有獨立庫存，互不干涉。

!!! tip "虛實庫存管理"

    建議在倉庫或門市後場將 **官網待出貨商品** 與 **門市待售商品** 實體分區擺放，以避免人員在揀貨或結帳時發生混淆。


### 選擇庫存管理模式

根據您的品牌營運需求，可選擇以下模式：

- **中心配發模式**：貨物進入 **EC 倉** 或指定 **POS 門市倉**，再由該倉分發至 EC 倉與各 POS 門市。
- **門市自主模式**：貨物統一進入 **第三方倉**，官網與各門市直接向第三方倉進貨，自行執行入庫作業。



## 庫存管理

### 管理 EC 庫存

=== "異動後可查找庫存調整記錄"

    如需有完整庫存調整紀錄，請使用以下方式調整庫存。

    #### 管理 EC、POS 全通路庫存

    1. 登入 POS 管理後台，前往 **POS 功能 > 全通路庫存管理**。
    2. 依據庫存異動類型，選擇操作方式：

        | 操作類型 | 庫存增減 | 執行角色 | 實務應用情境 |
        | ------- | -------- | ------- | ----------- | 
        | [出倉](出倉單) | 減少 | 出貨方 | 總倉配貨至子門市 |
        | [進倉](進倉單) | 增加 | 進貨方 | 採購到貨、進貨入庫 |
        | [調倉](調倉單) | 增加 | 缺貨方 | 缺貨方向庫存充足方提出庫存調撥支援 |
        | [盤點](庫存盤點) | 減少/增加 | 不限 | 定期清點後，依據實測數量修正系統帳面值<br>商品毀損報廢、消費者退貨回架、庫存異常時手動調減 | 

=== "異動後不可查找庫存調整記錄"

    採取以下方式更新 EC 庫存，系統將無法回溯調整紀錄，且異動資訊不會反映於常規報表中。
    
    若後續有追溯原始資料之需求，請洽詢 CYBERBIZ 客服協助；相關服務將依調閱之資料量與時間區間，酌收技術作業費用。

    === "於商品列表頁單筆編輯"

        1. 登入 POS 管理後台，前往 **商品 > 所有商品**。
        2. 直接修改商品 **庫存數量** 欄位。

    === "於商品明細頁單筆編輯"

        1. 登入 POS 管理後台，前往 **商品 > 所有商品 > 點擊指定商品**
        2. 直接修改商品 **庫存數量** 欄位。

    === "使用Excel匯入功能批次編輯"
    
        1. 登入 POS 管理後台，前往 **商品 > Excel 大量匯入商品**。
        2. 請先 [匯出指定商品]()，於 Excel 中編輯庫存欄位，再 [操作匯入]()，即可批量編輯。


### 管理 POS 庫存

=== "管理 EC、POS 全通路庫存"

    登入 POS 管理後台，前往 **POS功能 > 全通路庫存管理**。


=== "管理指定 POS 門市庫存"


    === "於後台管理"

        登入 POS 管理後台，前往 **POS功能 > 所有POS商店 > 選擇指定門市 > 庫存管理**。

    === "於前台管理"

        登入 POS 前台，前往 **庫存**。

         > 僅店長可進行操作，店員無此權限。

依據庫存異動類型，選擇操作方式：

| 操作類型 | 庫存增減 | 執行角色 | 實務應用情境 |
| ------- | -------- | ------- | ----------- | 
| [出倉](出倉單) | 減少 | 出貨方 | 總倉配貨至子門市 |
| [進倉](進倉單) | 增加 | 進貨方 | 採購到貨、進貨入庫 |
| [調倉](調倉單) | 增加 | 缺貨方 | 缺貨方向庫存充足方提出庫存調撥支援 |
| [盤點](庫存盤點) | 減少/增加 | 不限 | 定期清點後，依據實測數量修正系統帳面值<br>商品毀損報廢、消費者退貨回架、庫存異常時手動調減 | 


## 初始庫存建置

若您已在使用 EC 官網系統，在正式啟用 POS 後，請務必完成以下 **庫存初始化** 步驟，以確保虛實通路帳目獨立且準確：

1. **校正 EC 線上庫存**：
    請先確保官網現有庫存數據正確。您可以參考 [管理 EC 庫存](全通路庫存管理/#管理-EC-庫存)，選擇適合您的更新方式。
2. **建立 POS 門市首期庫存**：
    針對實體店面現有的商品數量，請採取 [第三方進倉](進倉單/#第三方進倉作業) 方式進行初始化。

完成初始入庫後，後續的日常進銷存管理，請參閱 [管理 POS 庫存](全通路庫存管理/#管理-POS-庫存)，選擇適合您的更新方式。
