# 設定商品配送屬性（一般宅配）
綁定一般宅配商品的配送物流、溫層及出貨通路
{ .page-subtitle } 

[:material-arrow-right-circle:](){ .badge-icon title="相關操作"}[[設定商品配送屬性（宅配貨到付款）]]{ .badge-text }

---

## 說明概述
=== ":material-information-outline: 注意事項"

	- [ ] 當購物車內的商品屬於不同「配送屬性」時，系統將自動拆分成[[設定多購物車#多購物車結帳說明|多購物車結帳]]，並個別計算優惠與運費。
	- [ ] 商品綁定後新增物流時，請檢查[[#商品綁定後新增配送物流|僅限特定物流的商品]]設定，避免漏綁。
	
=== ":material-lightbulb-outline: 應用情境"

	- [ ] 配送物流綁定：當商品需指定特定物流（如黑貓、宅配通、郵局等）配送時，可彈性設定。
	- [ ] 配送溫層綁定：針對需特殊溫層（如常溫、冷藏、冷凍）配送的商品，可設定多溫層選項。
	- [ ] 出貨通路綁定：當商品分屬不同出貨地點（如不同倉庫、門市或廠商）時，可設定商品通路。

---

## 操作流程
### 步驟一：設定物流不同溫層的運費

首先，您需要在後台設定各物流方式及其對應溫層的運費規則。


=== ":material-numeric-1-circle: 前往物流設定頁面"
	
	- [ ] 登入 CYBERBIZ 電商後台，前往「金物流」→「宅配物流」。
	- [ ] 點擊「自訂物流」分頁。

=== ":material-numeric-2-circle: 新增並設定運費規則"
	- [ ] 點擊「新增自訂物流」。
	- [ ] 輸入「物流名稱」。

	??? warning "請勿重複建立「黑貓貨到付款」與「宅配通貨到付款」"
		系統已預設提供「黑貓貨到付款」與「宅配通貨到付款」物流選項（`企業版`支援黑貓 / 宅配通貨到付款），**請勿再次建立相同名稱的運送方式**，以避免產生結帳錯誤或物流判斷異常。如您仍需建立這兩個選項，可輸入其他名稱代替或在名稱前後加上其他文字以示區別。

	- [ ] 選擇「運送地區」及「運送溫層」（預設為常溫）。
	- [ ] 在「訂單金額運費設定」區塊，輸入「消費金額」與對應的「運費」。
	- 若要設定不同消費金額的運費門檻，請再次點擊「新增設定」。

### 步驟二：設定商品配送溫層

完成物流運費設定後，您可以為個別商品綁定其適用的運送溫層。

- [ ] 登入 CYBERBIZ 電商後台，前往「商品」→「所有商品」。
- [ ] 點擊欲設定的商品，進入編輯頁面。
- [ ] 點擊「設定」頁籤。
- [ ] 在「溫層和物流配送設定」區塊中，勾選此商品綁定運送的「溫層」（可複選）。預設為常溫。

![溫層配送物流設定](https://www.cyberbiz.co/support/wp-content/uploads/2020/07/溫層配送物流設定-一般宅配8.png){ .screenshot }

### 步驟三：設定商品配送方式（物流）

您可以為個別商品綁定其可用的配送物流方式。

- [ ] 登入 CYBERBIZ 電商後台，前往「商品」→「所有商品」。
- [ ] 點擊欲設定的商品，進入編輯頁面。
- [ ] 在商品編輯頁面中，找到「溫層和物流配送設定」區塊。
- [ ] 「物流綁定狀態」`ON` 代表可使用該快遞/物流配送；`OFF` 則代表此商品不適用於該快遞/物流配送。
- 商品預設為[[#步驟一：設定物流不同溫層的運費|全部物流選項]]皆綁定（即所有物流皆可配送）；至少綁定一種配送物流。

![溫層配送物流設定](https://www.cyberbiz.co/support/wp-content/uploads/2019/03/溫層配送物流設定-一般宅配9.png){ .screenshot }

- [ ] 確認修改提醒：首次修改綁定狀態時，會跳出設定提醒。請閱讀完注意事項後，勾選並點擊「確認」。若未確認提醒，每次修改物流綁定都會跳出提醒。

![溫層配送物流設定](https://www.cyberbiz.co/support/wp-content/uploads/2019/03/溫層配送物流設定-一般宅配10.png)

### 步驟四：修改物流運費設定的注意事項
當您修改物流運費設定時，請留意以下事項：

=== "變更計算運費基準"
    若您變更計算運費的基準，必須重新設定物流的溫層、運費規則，以及商品頁面綁定配送物流的設定。

    -   商品會**預設為適用所有配送方式**。如需要綁定特定物流的商品，記得個別去設定。
    -   商品頁面中快遞/物流名稱也會變更為目前您於「物流運費設定」建立的項目。
    
    ![溫層配送物流設定](https://www.cyberbiz.co/support/wp-content/uploads/2019/03/溫層配送物流設定-一般宅配5.png){ .screenshot }
    ![溫層配送物流設定](https://www.cyberbiz.co/support/wp-content/uploads/2020/07/溫層配送物流設定-一般宅配6.png){ .screenshot }

=== "變更物流方式名稱"
    若您變更物流方式的名稱，原本商品綁定的物流都會被取消綁定（以物流名稱綁定商品，如物流名稱變更，舊的物流名稱即不存在），請重新[[#步驟三：設定商品配送方式（物流）|綁定物流配送]]方式。

    > 舉例：商品 A 原本綁定用「黑貓」配送，若您變更物流名稱將「黑貓」改成「黑貓宅急便」，請至商品 A 的商品頁面將物流配送方式重新綁定「黑貓宅急便」。

## 情境範例
### 商品綁定後新增配送物流

若您在完成設定商品綁定物流後，又新增了新的配送物流：

- :material-check: **原本「適用所有配送物流」的商品** ➜ 會自動綁定新物流（仍符合適用所有物流）。
- :material-alert-outline: **原本「只綁定特定配送物流」的商品** ➜ 不會綁定新物流，如需綁定請至商品頁面設定。

| 商品 | 原物流設定 | 新增物流後結果 |
|---|---|---|
| A | 適用黑貓＋宅配通（全部物流） | :material-check: 自動新增新竹物流 |
| B | 僅適用黑貓 | :material-close: 不自動新增新竹物流（仍僅黑貓） |
	
!!! tip "新增物流後，請檢查特定物流綁定商品，避免漏綁。"

### 不同配送物流、溫層、出貨通路

=== "後台設定"
    - [ ] 物流運費設定：設定黑貓、宅配通兩種物流配送方式，商品設定頁面就會有此兩種物流可選擇綁定。
    - [ ] 溫層配送：已內建常溫、冷藏、冷凍三種，商品設定頁面會有此三種物流可選擇綁定。
    - [ ] 商品頁面設定：設定台北門市、台中門市兩個商品通路（出貨地點）。
	
	??? tip "簡化商品「配送屬性」設定以降低棄單風險"
		過多的商品配送屬性（商品通路、溫層、配送物流）會導致系統在結帳時自動將商品拆分至多個購物車。每個購物車需獨立結帳，並生成各自的訂單，且優惠與運費將分別計算。此情況會增加結帳步驟與運費成本，可能導致顧客棄單。建議僅在必要時設定多種配送屬性，以簡化購物流程並提升結帳體驗。
		
=== "前台呈現"
    當顧客購買了不同配送物流、溫層、出貨通路的商品時，前台畫面呈現如下：

    - [ ] 最上方會列出不同的出貨通路，顧客先選擇一種出貨通路，再選擇該通路中其中一個配送溫層，溫層中再選擇配送物流，進行結帳。
    - [ ] 剩餘未結帳的商品會保留在購物車中等待結帳。
    - [ ] 顧客須完成多個購物車的結帳，建立多筆訂單（若未達各購物車免運門檻，則須支付多筆運費）。
    ![不同溫層配送通路舉例](https://www.cyberbiz.co/support/wp-content/uploads/2019/03/不同溫層配送通路舉例.png){ .screenshot }

## 常見問題

??? question "如果我修改了物流名稱，商品綁定會失效嗎？"
    是的，若您變更物流方式的名稱，原本商品綁定的物流都會被取消綁定。您需要到商品頁面重新綁定物流配送方式。

??? question "我可以設定多個溫層給一個商品嗎？"
    可以。在商品編輯頁面的「設定」頁籤中，您可以勾選多個溫層來綁定商品。

??? question "為什麼顧客結帳時會出現多個購物車？"
    當顧客購買的商品包含不同溫層、配送物流或出貨通路時，系統會自動將這些商品分屬在獨立的購物車中，每個購物車需獨立結帳。瞭解[[設定多購物車#多購物車結帳說明|更多]]。
